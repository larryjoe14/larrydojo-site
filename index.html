The LarryDoJo workflow is a tight loop between Claude, GitHub, and Netlify. GitHub is the source of truth. Netlify auto-deploys on every commit. Edits ship in minutes, not weeks.

That means continuous updates without retainer bloat. AEO and SEO optimization gets baked into the way the site is built, not bolted on after launch.

  • → Single source of truthEvery site lives in GitHub. No drift, no mystery zips, no "which version is live."
  • → Auto-deploy on commitNetlify pushes changes live in about 30 seconds. Client sees the new URL the same day.
  • → AEO + SEO from day oneStructured for AI answer engines and traditional search both. Built in, not retrofitted.
  • → Low monthly overheadStatic-first hosting. No per-seat licenses. Predictable costs that scale with you.
The Workflow
7 steps, every round, every client
  1. Receive client feedback. PDF, email, Slack, doesn't matter.
  2. Pull current code from GitHub (the canonical version).
  3. Refresh project knowledge with Claude, fresh every round.
  4. Share the feedback. Claude confirms with a numbered checklist.
  5. Output changed files, ready to commit.
  6. Push to GitHub. Commit message logs the round.
  7. Netlify auto-deploys in about 30 seconds. Send the live URL.
